HYD VNTG is a responsive storefront prototype built around one honest, complete workflow. The catalog is synthetic, the product images are stored locally, and the default build needs no account or external service. There is no payment processor, checkout, inventory reservation, shipping, fulfillment, or production order backend. The point of this demonstration is to show how a storefront can remain useful while external boundaries fail closed.

The home experience presents a focused Hyderabad-inspired visual system, featured items, prices, and stock labels. These names and quantities are demo fixtures, not real offers for sale. Because the six catalog images live inside the repository, browsing does not depend on an image CDN. Category filters update the visible collection immediately, and product cards expose their order control to pointer and keyboard focus.

I will select the Black 705 Tee from Vintage Tees. The product opens in a labeled dialog, moves keyboard focus to the close control, supports Escape, and keeps the content scrollable on a short viewport. I can choose size L and enter the documented synthetic buyer details. The phone field accepts ten to fifteen digits. These values stay in component memory and are not intentionally stored.

In the default build, WhatsApp has no destination. Submitting therefore generates this complete local message preview and labels it not sent. No popup opened and no placeholder phone number was contacted. If an owner later configures a verified international-digits destination, the same deterministic message builder can create a WhatsApp deep link. Instagram is also visibly unavailable rather than linking to a generic account.

The admin route has three explicit states: disabled, browser-only local demo, or Supabase. With no configuration, a sign-in attempt fails closed and stale browser sessions are removed. Local demo credentials are browser-visible build values, so they can never protect real inventory. In Supabase mode, the client accepts only a server-assigned app metadata admin role. The example row-level policies enforce that same role for insert, update, and delete. A failed Supabase mutation surfaces as failure and never silently becomes a local write. This repository does not claim those policies are deployed to a real project.

The verified evidence includes unit coverage for destination validation, message construction, catalog recovery and state transitions, local session rejection, admin-role checks, and policy structure. Browser tests cover this desktop preview, the rejected admin login, mobile navigation, visible focus, and horizontal overflow. The production build and dependency audit also pass.

HYD VNTG remains a portfolio prototype. Its strongest claim is a responsive, deterministic catalog and safe order-preparation flow—not a live store. The private duplicate remains untouched, and only this public repository is the canonical portfolio entry. HYD VNTG by badugujashwanth-create.
